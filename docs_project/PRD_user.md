Этот документ детально описывает функционал, который необходимо разработать для покрытия всех потребностей Покупателя (Guest/Registered Buyer) на платформе Market 365. Вы можете использовать этот PRD для создания конкретных задач (тасок) в трекере.

---

# PRD: Модуль «Покупатель» (Buyer Module)

## 1. Цель модуля
Обеспечить пользователю максимально быстрый, интуитивно понятный и безопасный путь от формирования потребности (запроса) до установления контакта с продавцом. Модуль должен удерживать пользователя на платформе и стимулировать возвраты (retention).

## 2. Пользовательские роли
* **Гость (Unregistered Buyer):** Доступен поиск, просмотр карточек товаров, просмотр профилей продавцов. Скрыты контакты, недоступен чат, нельзя добавлять в Избранное.
* **Авторизованный покупатель (Registered Buyer):** Полный доступ ко всем функциям взаимодействия (чаты, телефоны, подписки, жалобы).

---

## 3. Функциональные требования (Эпики и Сториз)

### 3.1. Поиск и навигация (Search & Discovery)
Ключевой механизм платформы. Должен работать быстро и выдерживать сложные запросы.

* **Глобальный поиск:**
    * Поиск по ключевым словам в заголовках и описаниях объявлений.
    * Подсказки при вводе (Autocomplete) на основе популярных запросов и категорий.
* **Дерево категорий и «Хлебные крошки»:**
    * Покупатель может перемещаться по вложенным категориям (например, `Главная > Транспорт > Авто > Легковые`).
    * На каждой странице отображаются хлебные крошки для возврата на уровень выше.
* **Умные фильтры (Dynamic Attributes):**
    * Фильтры должны динамически меняться в зависимости от выбранной категории (для телефонов — память и цвет, для авто — пробег и год выпуска).
    * Фильтр по состоянию (Новое / Б/у).
    * Слайдер диапазона цен (от и до, в GBP - £).
* **Локальный поиск (Geo-location):**
    * Фильтрация по городу или британскому Postcode.
    * Настройка радиуса поиска (например, +5, +10, +50 миль).
    * Переключение выдачи: Списком / Сеткой (Галерея) / На карте.
* **Сортировка:**
    * По релевантности, от дешевых к дорогим, от дорогих к дешевым, по дате добавления.

### 3.2. Карточка объявления (Listing Page)
Страница, где покупатель принимает решение о контакте.

* **Медиа-галерея:** * Просмотр фотографий с возможностью увеличения (Lightroom).
    * Свайп фото на мобильных устройствах; миниатюры или стрелки на десктопе.
* **Информационный блок:**
    * Заголовок, Цена, Локация, Дата публикации, Количество просмотров.
    * Блок динамических характеристик (таблица или список атрибутов товара).
    * Текстовое описание.
* **Блок рекомендаций (Cross-sell):**
    * Секция «Похожие объявления» под описанием товара (алгоритм подбирает товары из той же подкатегории в похожем ценовом диапазоне).

### 3.3. Взаимодействие и коммуникация (Engagement)
Связь с продавцом — главная конверсионная метрика для покупателя.

* **Показать номер телефона:**
    * Номер продавца скрыт маской (например, `+44 791...`).
    * По клику: проверка авторизации -> раскрытие номера -> фиксация клика в БД для статистики продавца.
* **Внутренний чат:**
    * Кнопка «Написать сообщение».
    * Открытие интерфейса чата (Real-time). В заголовке чата должна быть закреплена мини-карточка товара (фото, название, цена), чтобы контекст не терялся.
    * Статусы прочтения сообщений (отправлено, доставлено, прочитано).
* **Мини-профиль продавца:**
    * Блок на карточке товара: Имя продавца, аватар, дата регистрации, средний рейтинг.
    * Кнопка «Все объявления автора» (переход на список активных лотов конкретного пользователя).

### 3.4. Удержание и персонализация (Retention)
Инструменты для отложенного спроса.

* **Избранное (Favorites):**
    * Добавление объявления в закладки кликом на иконку «Сердечко».
    * Отображение бейджа с количеством сохраненных товаров в шапке профиля.
    * *Уведомление о снижении цены:* Если продавец снижает цену на товар из Избранного, покупателю уходит пуш/email.
* **Сохраненные поиски (Saved Searches):**
    * Кнопка «Подписаться на новые» в результатах поиска по фильтрам.
    * Рассылка уведомлений, когда на платформе появляется новый товар, соответствующий сохраненным фильтрам.
* **Шеринг:**
    * Генерация короткой ссылки на товар с правильными Open Graph тегами (чтобы в WhatsApp/Telegram корректно подтягивалось превью фото и цены).

### 3.5. Безопасность (Trust & Safety)
* **Репортинг (Жалобы):**
    * Кнопка «Пожаловаться на объявление» с выпадающим списком причин (мошенничество, спам, неверная категория).
* **Система отзывов:**
    * После завершения сделки покупатель может оставить оценку (от 1 до 5 звезд) и текстовый комментарий продавцу.

## 4. Архитектурные и Технические требования

Для реализации данного модуля потребуется надежная архитектура БД и быстрый бэкенд.

* **База данных (PostgreSQL):** Идеально подходит для реализации динамических атрибутов (Умных фильтров). Категорийные характеристики товаров целесообразно хранить в формате `JSONB`, что позволит гибко индексировать и фильтровать уникальные поля без необходимости создавать десятки лишних таблиц.
* **Поисковый движок:** Для обеспечения быстрого полнотекстового поиска (с опечатками и синонимами) рекомендуется рассмотреть интеграцию Elasticsearch или использование встроенных полнотекстовых возможностей PostgreSQL (Full Text Search).
* **Real-time Чат:** Функционал внутреннего мессенджера требует работы через WebSockets. Для обработки множества одновременных подключений чата оптимально выделить отдельный микросервис (например, написанный на Node.js или Go), который будет асинхронно обрабатывать сообщения и статусы прочтения, снижая нагрузку на основной монолит или API.
* **Frontend UI:** Адаптивная верстка (Mobile-first). Скелетные загрузки (Skeleton loaders) для карточек товаров во время выполнения поисковых запросов, чтобы интерфейс казался быстрее. Плавное переключение фильтров без жесткой перезагрузки страницы.

## 5. Критерии приемки (Definition of Done)
1.  Гость не может написать в чат или увидеть полный номер телефона без регистрации.
2.  При смене категории в поиске динамически меняется набор доступных фильтров.
3.  Поиск работает корректно и возвращает релевантные результаты.
4.  Сообщения во внутреннем чате доставляются без перезагрузки страницы.
5.  Добавление в Избранное корректно отрабатывает и сохраняется после перезахода в аккаунт.
6.  Все ссылки из "хлебных крошек" ведут на правильные страницы категорий.
