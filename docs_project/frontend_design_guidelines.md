# Frontend Design Guidelines
## Проект: Market 365

> Этот документ является обязательным стандартом для всех разработчиков и дизайнеров, работающих над платформой Market 365. Все новые компоненты, страницы и правки должны соответствовать описанным здесь правилам.

---

## 1. Принципы и философия дизайна

1. **Mobile First** — верстка начинается с мобильного разрешения (320px), затем расширяется до планшетного и десктопного. Ни один элемент не должен ломаться на экранах шириной менее 320px.
2. **Clarity over Cleverness** — интерфейс должен быть понятен с первого взгляда. Никаких скрытых жестов и неочевидных паттернов.
3. **Speed Perception** — интерфейс должен казаться быстрым. Skeleton loaders, оптимистичные обновления UI, отсутствие блокирующих спиннеров.
4. **Trust by Design** — визуальный язык платформы должен внушать доверие: чистые формы, четкая иерархия, явные статусы и подтверждения действий.

---

## 2. Логотип

> Жесткое требование, утвержденное заказчиком.

- Слово **«Market»** — располагается **сверху**.
- Число **«365»** — располагается **снизу**.
- Логотип не масштабируется произвольно: соотношение сторон фиксировано.
- Минимальный размер логотипа в шапке: `32px` по высоте.
- Логотип всегда кликабелен и ведет на главную страницу (`/`).
- Запрещено: растягивать, переворачивать, менять цвета логотипа без явного согласования.

## 3. Цветовая палитра

### Основные цвета
| Токен | HEX | Применение |
| :--- | :--- | :--- |
| `--color-primary` | `#FF6B00` | CTA-кнопки, акценты, активные состояния |
| `--color-primary-hover` | `#E05A00` | Hover-состояние primary кнопок |
| `--color-secondary` | `#1A1A2E` | Заголовки, основной текст |
| `--color-bg` | `#F5F5F5` | Фон страниц |
| `--color-surface` | `#FFFFFF` | Карточки, модальные окна, панели |
| `--color-border` | `#E0E0E0` | Разделители, границы полей |

### Семантические цвета
| Токен | HEX | Применение |
| :--- | :--- | :--- |
| `--color-success` | `#2E7D32` | Успешные действия, статус «Активно» |
| `--color-warning` | `#F57C00` | Предупреждения, статус «На модерации» |
| `--color-error` | `#C62828` | Ошибки, статус «Отклонено» |
| `--color-info` | `#1565C0` | Информационные сообщения |
| `--color-muted` | `#757575` | Вторичный текст, плейсхолдеры |

### Правила использования цвета
- `--color-primary` используется **только** для главного действия на экране (не более одной primary-кнопки в видимой области).
- Фон страницы всегда `--color-bg`, карточки — `--color-surface`.
- Текст на цветном фоне должен соответствовать контрастности **WCAG AA** (минимум 4.5:1 для обычного текста).

## 4. Типографика

### Шрифт
- **Основной шрифт:** `Inter` (Google Fonts) — для всего UI.
- **Запасной стек:** `system-ui, -apple-system, sans-serif`.

### Шкала размеров
| Токен | Размер | Line-height | Применение |
| :--- | :--- | :--- | :--- |
| `--text-xs` | `12px` | `1.4` | Метки, бейджи, подписи |
| `--text-sm` | `14px` | `1.5` | Вторичный текст, описания |
| `--text-base` | `16px` | `1.6` | Основной текст, поля форм |
| `--text-lg` | `18px` | `1.5` | Подзаголовки |
| `--text-xl` | `22px` | `1.4` | Заголовки секций |
| `--text-2xl` | `28px` | `1.3` | Заголовки страниц |
| `--text-3xl` | `36px` | `1.2` | Цена на карточке товара (десктоп) |

### Правила
- Минимальный размер текста в интерфейсе — `12px`.
- Основной текст страницы — `16px`, `font-weight: 400`.
- Заголовки — `font-weight: 600` или `700`.
- Цена объявления — `font-weight: 700`, цвет `--color-secondary`.
- Не использовать более 3 разных размеров шрифта на одном экране.

## 5. Сетка и отступы

### Брейкпоинты
| Название | Ширина | Описание |
| :--- | :--- | :--- |
| `xs` | `< 480px` | Маленькие смартфоны |
| `sm` | `480px – 767px` | Смартфоны |
| `md` | `768px – 1023px` | Планшеты |
| `lg` | `1024px – 1279px` | Небольшие ноутбуки |
| `xl` | `≥ 1280px` | Десктоп |

### Контейнер
- Максимальная ширина контента: `1280px`.
- Горизонтальные отступы контейнера: `16px` (mobile), `24px` (tablet), `32px` (desktop).

### Базовая единица отступов (8px grid)
Все отступы (`margin`, `padding`, `gap`) кратны `8px`:
`4px / 8px / 12px / 16px / 24px / 32px / 48px / 64px`

### Сетка карточек объявлений
| Брейкпоинт | Количество колонок |
| :--- | :--- |
| `xs` | 1 |
| `sm` | 2 |
| `md` | 3 |
| `lg` | 3–4 |
| `xl` | 4–5 |

## 6. Компоненты

### 6.1. Кнопки

| Вариант | Применение |
| :--- | :--- |
| **Primary** | Главное действие на экране (Опубликовать, Купить, Написать) |
| **Secondary** | Второстепенное действие (Редактировать, Сохранить черновик) |
| **Ghost / Outline** | Третичные действия, деструктивные с подтверждением |
| **Danger** | Необратимые действия (Удалить объявление) |
| **Link** | Навигационные действия внутри текста |

**Правила кнопок:**
- Минимальная высота кнопки: `44px` (touch target).
- Минимальная ширина: `88px`.
- Скругление: `8px` (стандарт), `4px` (компактные), `100px` (pill — только для тегов/бейджей).
- Состояния: `default`, `hover`, `active`, `disabled`, `loading` (спиннер внутри кнопки).
- Кнопка в состоянии `loading` — некликабельна, показывает inline-спиннер.
- Не использовать более одной `Primary` кнопки в одной секции.

### 6.2. Карточка объявления (Listing Card)

Обязательные элементы карточки:
- **Фото** — соотношение сторон `4:3` (cover), с placeholder-изображением при отсутствии фото.
- **Цена** — крупно, `font-weight: 700`. Для «Бесплатно» — зеленый цвет (`--color-success`).
- **Заголовок** — максимум 2 строки, с `text-overflow: ellipsis`.
- **Локация + дата** — мелко, `--color-muted`.
- **Иконка «Избранное»** (сердечко) — в правом верхнем углу фото, всегда доступна.

Опциональные элементы (для платного продвижения):
- Бейдж **«ТОП»** — оранжевый (`--color-primary`), левый верхний угол фото.
- Выделение цветом — тонкая цветная рамка вокруг карточки (`2px solid --color-primary`).

### 6.3. Формы и поля ввода

- Высота поля: `48px` (стандарт), `40px` (компактный).
- Label всегда над полем, не внутри (placeholder — только подсказка, не замена label).
- Состояния: `default`, `focus` (синяя обводка), `error` (красная обводка + текст ошибки снизу), `disabled` (серый фон), `success`.
- Текст ошибки — `--text-sm`, `--color-error`, появляется под полем.
- Обязательные поля помечаются `*` после label.
- Многошаговые формы (публикация объявления) — прогресс-бар или нумерованные шаги сверху.

### 6.4. Модальные окна и диалоги

- Фон затемняется оверлеем (`rgba(0,0,0,0.5)`).
- Максимальная ширина: `560px` (стандарт), `800px` (расширенный).
- Закрытие: кнопка `×` в правом верхнем углу + клик по оверлею + клавиша `Esc`.
- Деструктивные действия (удаление) — всегда через диалог подтверждения с явной кнопкой `Danger`.

### 6.5. Уведомления и Toast

- Позиция: правый нижний угол (desktop), снизу по центру (mobile).
- Автоскрытие: `4000ms` для success/info, `6000ms` для error (с кнопкой закрытия).
- Типы: `success`, `error`, `warning`, `info`.

### 6.6. Skeleton Loaders

Обязательны для:
- Сетки карточек объявлений при загрузке поиска.
- Детальной страницы объявления.
- Списка сообщений в чате.

Правила: цвет `#E0E0E0` с анимацией shimmer (`background: linear-gradient(...)`), форма повторяет реальный контент.

### 6.7. Бейджи и статусы

| Статус | Цвет фона | Цвет текста |
| :--- | :--- | :--- |
| Активно | `#E8F5E9` | `#2E7D32` |
| На модерации | `#FFF3E0` | `#F57C00` |
| Отклонено | `#FFEBEE` | `#C62828` |
| Неактивно / Архив | `#F5F5F5` | `#757575` |
| ТОП | `#FF6B00` | `#FFFFFF` |
| Verified Business | `#E3F2FD` | `#1565C0` |

---

## 7. Иконки

- Библиотека: **Lucide Icons** (или аналогичная с MIT-лицензией).
- Размеры: `16px` (inline), `20px` (стандарт в UI), `24px` (акцентные).
- Цвет иконок наследуется от цвета текста родительского элемента.
- Иконки без текстовой подписи — обязательный `aria-label` или `title`.

---

## 8. Изображения и медиа

- **Соотношение сторон карточки:** `4:3` — обязательно через `aspect-ratio: 4/3`.
- **Галерея на детальной странице:** главное фото `16:9` или `4:3`, миниатюры под ним.
- **Аватары:** всегда круглые (`border-radius: 50%`), размеры: `32px`, `40px`, `48px`, `64px`.
- **Placeholder при отсутствии фото:** серый фон + иконка камеры по центру.
- **Lazy loading:** атрибут `loading="lazy"` для всех изображений ниже fold.
- **Alt-текст:** обязателен для всех `<img>` тегов.

## 9. Навигация и шапка (Header)

### Десктоп
- Логотип слева.
- Глобальный поиск по центру (занимает ~40% ширины).
- Справа: кнопка «Разместить объявление» (Primary), иконки уведомлений, чата, профиля.

### Мобильный
- Логотип по центру или слева.
- Иконка поиска (открывает полноэкранный поиск).
- Нижняя навигационная панель (Tab Bar) с иконками: Главная, Поиск, Разместить (+), Сообщения, Профиль.
- Кнопка «Разместить» в Tab Bar — всегда выделена (`--color-primary`), круглая.

### Хлебные крошки (Breadcrumbs)
- Отображаются на страницах категорий и карточках товаров.
- Формат: `Главная > Транспорт > Авто > Легковые`.
- Последний элемент — некликабелен, цвет `--color-muted`.

## 10. Страницы и ключевые флоу

### 10.1. Главная страница
- Hero-блок с поисковой строкой.
- Сетка популярных категорий (иконки + название).
- Лента последних объявлений.
- Плейсхолдеры для баннеров Google AdSense (явно размечены в верстке комментарием `<!-- AdSense Placeholder -->`).

### 10.2. Страница поиска / категории
- Фильтры — слева (desktop) или в drawer/bottom sheet (mobile).
- Переключатель вида: Список / Сетка / Карта.
- Сортировка — выпадающий список над результатами.
- Пагинация или infinite scroll (согласовать с командой).

### 10.3. Карточка объявления (детальная)
- Галерея фото — вверху, с возможностью открытия в lightbox.
- Цена + кнопки «Написать» и «Показать номер» — sticky-блок на мобильном (прилипает к низу экрана).
- Блок характеристик — таблица или список пар «ключ: значение».
- Мини-профиль продавца — справа (desktop) или под описанием (mobile).
- Секция «Похожие объявления» — внизу страницы.

### 10.4. Личный кабинет (Dashboard)
- Вкладки: Активные / На модерации / Неактивные / Черновики.
- Каждое объявление в списке — горизонтальная карточка с фото, заголовком, статусом, счетчиками и кнопками действий.
- Кошелек/баланс — отдельная вкладка с историей транзакций.

### 10.5. Форма публикации объявления
- Многошаговый флоу: Категория → Характеристики → Фото → Цена и локация → Продвижение (upsell) → Публикация.
- Прогресс-бар сверху.
- Автосохранение черновика каждые `30 секунд` (индикатор «Сохранено» рядом с прогресс-баром).
- Drag-and-drop зона для фото с превью и возможностью перетаскивания для изменения порядка.

### 10.6. Чат (Мессенджер)
- Двухколоночный layout (desktop): список диалогов слева, переписка справа.
- Мобильный: список диалогов → переход в переписку (отдельный экран).
- В шапке диалога — мини-карточка товара (фото + название + цена).
- Статусы сообщений: отправлено (одна галочка), доставлено (две галочки), прочитано (две синие галочки).

### 10.7. B2B Мини-сайт (Витрина магазина)
- Header витрины: обложка (banner) на всю ширину, поверх — логотип магазина, название, рейтинг, бейдж «Verified Business».
- Внутренний поиск фильтрует только товары этого магазина.
- Сетка товаров — стандартные карточки объявлений.

## 11. Анимации и переходы

- Стандартная длительность переходов: `200ms` (hover), `300ms` (открытие/закрытие элементов).
- Easing: `ease-in-out` для большинства переходов.
- Hover на карточках: легкое поднятие (`transform: translateY(-2px)`) + усиление тени.
- Не использовать анимации длиннее `500ms` для функциональных элементов.
- Уважать системный параметр `prefers-reduced-motion`: при его активации все анимации отключаются.

---

## 12. Доступность (Accessibility)

- Контрастность текста — минимум **WCAG AA** (4.5:1 для обычного, 3:1 для крупного).
- Все интерактивные элементы доступны с клавиатуры (`Tab`, `Enter`, `Space`, `Esc`).
- Фокус-кольцо (`outline`) — видимое, не скрывать через `outline: none` без альтернативы.
- Семантическая HTML-разметка: `<nav>`, `<main>`, `<header>`, `<footer>`, `<article>`, `<section>`.
- Изображения — обязательный `alt`.
- Иконки без текста — `aria-label`.
- Формы — связка `<label for="...">` + `id` у поля.

## 13. Локализация и форматирование (UK Market)

- **Валюта:** Фунт стерлингов, символ `£` перед суммой. Пример: `£ 1,250`.
- **Разделитель тысяч:** запятая (`,`). Пример: `£ 12,500`.
- **Дата:** формат `DD/MM/YYYY`. Пример: `03/04/2026`.
- **Время:** 12-часовой формат с AM/PM. Пример: `2:30 PM`.
- **Телефон:** маска `+44 XXXX XXXXXX`. Неполный номер скрывается как `+44 791...`.
- **Postcode:** отображается в верхнем регистре. Пример: `SW1A 1AA`.
- Язык интерфейса: **английский (UK)**.

---

## 14. Производительность и оптимизация

- Изображения — форматы `WebP` с fallback на `JPEG`.
- Карточки объявлений — lazy loading изображений.
- Шрифты — `font-display: swap`, предзагрузка через `<link rel="preload">`.
- Критический CSS — инлайн в `<head>`, остальное — асинхронно.
- Целевые метрики Core Web Vitals: LCP < 2.5s, CLS < 0.1, INP < 200ms.

---

## 15. Запрещенные паттерны (Anti-patterns)

- Запрещено использовать `!important` в CSS без крайней необходимости.
- Запрещено скрывать `outline` без замены на видимый фокус-индикатор.
- Запрещено использовать `<div>` вместо семантических тегов там, где они применимы.
- Запрещено показывать полный номер телефона незарегистрированным пользователям.
- Запрещено использовать более одной `Primary`-кнопки в одной видимой секции.
- Запрещено анимировать элементы длиннее `500ms` без уважительной причины.
- Запрещено менять логотип (цвет, пропорции, порядок «Market» / «365») без согласования.
